# Button

The primary interaction element. Renders as a `<button>` by default; automatically becomes an `<a>` when `href` is provided and the button is not disabled or loading.

## Variants

| Variant | When to use |
|---------|-------------|
| `solid` | Primary actions with strong visual weight |
| `outline` | Secondary actions that need clear affordance |
| `ghost` | Low-priority actions within dense UI |
| `link` | Inline actions that flow with text |

## Accessibility

- Uses native `<button>` semantics; no additional ARIA needed.
- `disabled` buttons are excluded from focus order and have `pointer-events: none`.
- Loading state hides the label with `sr-only` and exposes `loadingText` to screen readers.
- Focus ring is always visible via `focus-visible:ring`.
